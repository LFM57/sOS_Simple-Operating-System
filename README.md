# sOS (Simple Operating System)

sOS is a 32-bit (i686) hobbyist Operating System built from scratch. It features preemptive multitasking, a custom FAT16 filesystem implementation with a rudimentary user/permission system, Ring 3 user-space applications (ELF loading), and a built-in shell with various utilities, including a basic text editor (`nano`-lite).

For a deep dive into the inner workings of sOS, please refer to [TECHNICAL_DETAILS.md](TECHNICAL_DETAILS.md).

## Features
* **Preemptive Multitasking:** Context switching driven by the PIT (Programmable Interval Timer).
* **Ring 3 User Space:** Proper privilege separation. Applications run in user mode and communicate with the kernel via Syscalls (`int 0x80`).
* **ELF Binary Loader:** Capable of parsing and loading 32-bit ELF executables.
* **Custom FAT16 Filesystem:** Complete with read/write/append capabilities and directory traversal. Read/Write operations use the ATA PIO mode.
* **Multi-User & Permissions System:** Implements UIDs and permission masking using a previously reserved byte in the FAT16 directory entry. Passwords are hashed using the FNV-1a algorithm.
* **Built-in Shell:** Features auto-completion (Tab), command history (Arrows), typo suggestions, and background process execution (`&`).
* **Nano Light:** A basic built-in text editor to modify files directly inside the OS.

## System Requirements
sOS is extremely lightweight. To run it smoothly, you only need:
* **CPU:** 1 Core is largely sufficient.
* **RAM:** ~50 MB of RAM is recommended. (It can boot and run on as little as 15 MB, but be aware that it may crash under load due to out-of-memory errors).
* **Disk Space:** 16 MB (Default hard disk size generated by the Makefile).

## Prerequisites & Compilation

To build sOS, you will need a Linux environment (or WSL) with the following packages installed:
* `make`
* `grub-pc-bin` and `grub-rescue-pc` (for `grub-mkrescue`)
* `xorriso` (required by `grub-mkrescue` to build the ISO)
* `mtools` (for `mcopy`, `mmd`)
* `dosfstools` (for `mkfs.fat`)
* An `i686-linux-gnu-` cross-compiler toolchain (GCC, Binutils).

Clone the repository and simply run `make` at the root of the project:

```bash
make
```

This will compile the kernel and user apps, format a 16 MB FAT16 raw disk (`drive.img`), populate it, generate a bootable ISO (`sos.iso`), and convert the disk for VirtualBox (`drive.vdi`).

If you wish later on to clean the directory, you may run `make clean`. This will delete all compiled files, keeping only the source code.

## How to Launch sOS

### 1. Running with VirtualBox (Recommended)
VirtualBox is the recommended way to experience sOS. Follow these steps once you have generated the files:
1. You must first have "Expert" mode turned on. The easiest way to do so is to open the settings (`crtl + g`) and clic on the top left corner the "Expert" button. This should activate Expert Mode for all the application. You can now exit the parameters.
2. Assure you are on the home screen, and clic on the "New" button in the top left.
3. You will be prompted to enter informations:
    - In the "VM Name" field, input "sOS". 
    - Go straight to the "ISO Image" field and select the `sos.iso` file you just compiled (you can navigate in your computer file system by clicking the arrow on the right of the field, then "Other..."). This should prefill the remaining fields with "Other/Unknown" (or similar).
    - Clic on "Specify Virtual hardware" to open the sub-menu. Feel free to select as many ressources as you want, but sOS runs on a single core, and 64 MB is more than enough.
    - Last step, open the sub-menu "Specify virtual hard disk". You may see the "Create a new virtual hard disk" already selected. Uncheck it (VirtualBox generates a drive that's not compatible with sOS) and tic "Use an Existing Hard Disk File". Then, smash the folder icon on the right, and on the "Add" button at the top left corner of the window that just appeard to browse your computer and select the drive.vdi generated by the `make` command. 
4. You are all set! Just clic the "Start" green-arrow button at the top of the screen to launch sOS!

### 2. Running with QEMU (Legacy)
If you prefer QEMU for rapid testing, you can launch sOS with a single command (allocating 50MB of RAM):

```bash
qemu-system-i386 -m 50M -cdrom sos.iso -drive file=drive.img,format=raw,index=0,media=disk
```

### ⚠️ Deprecation Warning
**SSH over Serial Support is being phased out.** Support for interacting with the sOS shell over the COM1 serial port (IRQ 4) using an SSH-like terminal is deprecated. The legacy code is currently still in the kernel, but it will be entirely removed in upcoming versions to clean up the interrupt handler and input state machine.

## Default Credentials

The Makefile pre-populates the `/passwd` file with three default accounts:

| Username | Password | UID | Role |
|----------|----------|-----|------|
| `root`   | `root`   | 0   | Superuser |
| `user`   | `password` | 1   | Standard User (Good luck guessing it!) |
| `guest`  | *(Leave blank)*| 2   | Standard User (Just hit Enter) |

*Note: I completely forgot the password for the `user` account, but you can always log in as `root` and use the `passwd user` command to overwrite it!*

## Basic Commands

Once logged in, type `help` to see the available commands. Some useful ones include:
* `ls`, `cd`, `mkdir`, `rm`, `touch`, `cat`, `cp`, `mv`: Standard filesystem operations.
* `nano <file>`: Opens the Nano-lite text editor.
* `run <app.sos>`: Executes a Ring 3 ELF binary. Add `&` at the end to run it in the background.
* `ps`, `kill <pid>`: Process management.
* `su <user>`, `sudo <cmd>`: Switch user or execute as root.
* `chown`, `chmod`, `passwd`, `userlist`: User and permission management.
